治理说明
介绍
针对数据治理模型识别出的治理项，按照治理方案做相应治理。
治理方案
1. 存储–>hive–>hive空表
治理方案1: 表管理员后台对表执行drop表操作，命令为：‘drop table if exists table_name’。
治理方案2: 在我的治理列表中申请drop表操作。
2. 存储–>hive–>hive未管理生命周期
治理方案1: 表管理员后台对表执行设置生命周期操作，命令为:’alter table table_name set tblproperties(‘lifecycle’=
‘周期天’)’。
治理方案2: 在我的治理列表中申请设置生命周期操作。
3. 存储–>hive–>hive大表未压缩
治理方案: 对原表进行备份，然后创建压缩表，将原表数据迁移到新表中。
4. 存储–>hive–>Hive小文件过多
治理方案: 通过程序对表数据进程合并，同时要控制后续写入的文件大小。
5. 计算–>yarn–>yarn队列资源紧张
治理方案: 在yarn队列查询页面，查看该队列下占用资源比较多的任务，分析这些任务分配的资源是否合理;或者调整该队列
下各任务运行时间，尽量错峰运行或者联系资源管理员增加该队列的资源。
6. 计算–>yarn–>yarn队列资源过剩
治理方案: 在yarn队列查询页面，查看队列的历史使用率情况，若资源过多，可以将该队列的资源分配给其他资源紧张的队
列。
